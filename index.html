Pipeline Stage Details

GitHub Repository

The CI/CD pipeline begins with developers pushing code to a GitHub repository. This serves as the source control system where all code changes are tracked and managed.

Key Functions:

  • Source code management
  • Version control
  • Collaboration between developers
  • Pull requests and code reviews
  • Triggers the CI/CD pipeline on code changes

AWS CodeBuild

CodeBuild is a fully managed build service that compiles source code, runs tests, and produces software packages that are ready to deploy.

Key Functions:

  • Compiles and builds the application
  • Runs unit tests and integration tests
  • Creates deployment artifacts
  • Performs code quality checks
  • Generates build reports

AWS CodePipeline

CodePipeline is a continuous delivery service that automates the release process, enabling you to rapidly and reliably deliver features and updates.

Key Functions:

  • Orchestrates the entire CI/CD workflow
  • Manages transitions between stages
  • Provides visibility into pipeline status
  • Integrates with other AWS services
  • Supports manual approval steps when needed

AWS CodeDeploy

CodeDeploy automates application deployments to Amazon EC2 instances, on-premises instances, serverless Lambda functions, or Amazon ECS services.

Key Functions:

  • Automates deployment to target environments
  • Supports blue/green deployments
  • Handles rollbacks if deployments fail
  • Minimizes downtime during deployments
  • Provides deployment history and tracking

EC2 & Auto Scaling Group

Amazon EC2 provides resizable compute capacity in the cloud, while Auto Scaling Groups automatically adjust capacity to maintain performance and availability.

Key Functions:

  • Hosts the application on virtual servers
  • Automatically scales based on demand
  • Ensures high availability across multiple AZs
  • Replaces unhealthy instances
  • Integrates with load balancers for traffic distribution

Application Load Balancer (ALB)

ALB automatically distributes incoming application traffic across multiple targets, such as EC2 instances, in multiple Availability Zones.

Key Functions:

  • Distributes traffic evenly across instances
  • Performs health checks on targets
  • Supports path-based routing
  • Enables SSL/TLS termination
  • Provides high availability for applications

End Users

The final stage of the pipeline is delivering the application to end users who access it through the internet.

Key Points:

  • Users access the application through the ALB endpoint
  • Traffic is distributed across healthy EC2 instances
  • Users experience minimal downtime during updates
  • Application scales automatically to handle user load
  • Continuous delivery ensures users get the latest features quickly

How the CI/CD Pipeline Works

  1. Code Commit: A developer commits code changes to the GitHub repository, which triggers the CI/CD pipeline.
  2. Source Stage: AWS CodePipeline detects the change in the GitHub repository and pulls the latest code.
  3. Build Stage: CodeBuild compiles the code, runs tests, and packages the application into deployable artifacts.
  4. Deploy Stage: CodeDeploy takes the artifacts and deploys them to the target environment (EC2 instances in an Auto Scaling Group).
  5. Infrastructure: The application runs on EC2 instances within an Auto Scaling Group, which can scale based on demand.
  6. Load Balancing: The Application Load Balancer distributes traffic to healthy EC2 instances.
  7. User Access: End users access the application through the ALB endpoint, experiencing minimal downtime during updates.

Key AWS Services in the Pipeline

  • IAM (Identity and Access Management): Manages permissions for services to interact with each other securely.
  • CodeBuild: Compiles source code, runs tests, and produces software packages.
  • CodeDeploy: Automates application deployments to EC2 instances.
  • CodePipeline: Orchestrates the workflow between different stages of the pipeline.
  • EC2 (Elastic Compute Cloud): Provides virtual servers in the cloud to run applications.
  • Auto Scaling Group: Automatically adjusts capacity to maintain performance and availability.
  • ALB (Application Load Balancer): Distributes incoming application traffic across multiple targets.
  • CloudWatch: Monitors the application and infrastructure, collecting metrics and logs.
  • SNS (Simple Notification Service): Sends notifications about pipeline events and alerts.

Benefits of This CI/CD Pipeline

  • Automation: Reduces manual intervention and human error.
  • Speed: Accelerates the delivery of new features and bug fixes.
  • Reliability: Consistent and repeatable deployment process.
  • Quality: Automated testing ensures code quality before deployment.
  • Scalability: Infrastructure automatically scales with demand.
  • Visibility: Clear insights into the deployment process and application health.
  • Rollbacks: Easy recovery from failed deployments.
