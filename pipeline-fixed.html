Pipeline Stage Details

How This Secure CI/CD Pipeline Works

  1. Code Commit: Developers push code to Git, which triggers the pipeline.
  2. Static Analysis: Code is analyzed for quality and security issues without execution.
  3. CodeCommit to Static Analysis: Code is sent for thorough examination.
  4. Build & Test: AWS CodeBuild compiles the code and runs tests, including SAST.
  5. Staging Deployment: Application is deployed to a staging environment.
  6. Dynamic Application Security Testing: Security tools prepare to test the application.
  7. Test (DAST): Dynamic security testing is performed on the running application.
  8. Manual Approval: Team reviews test results and approves production deployment.
  9. Production Deployment: After approval, application is deployed to production.
  10. CloudWatch Events: Monitoring and notifications for the entire pipeline.

Security Features in This Pipeline

  • Static Application Security Testing (SAST): Analyzes source code for security vulnerabilities.
  • Dynamic Application Security Testing (DAST): Tests running applications for security issues.
  • Dependency Checking: Identifies vulnerabilities in third-party libraries.
  • Manual Approval Gate: Ensures human review of security findings before production.
  • Audit Logging: Tracks all changes and actions for compliance and security analysis.

AWS Services Used

  • CodePipeline: Orchestrates the entire CI/CD workflow.
  • CodeCommit: Manages source code repositories.
  • CodeBuild: Compiles code and runs tests.
  • CodeDeploy: Automates application deployment.
  • Elastic Beanstalk: Manages application environments.
  • CloudWatch: Monitors applications and infrastructure.
  • CloudTrail: Logs API activity for audit purposes.
  • AWS Config: Tracks resource configurations for compliance.
  • S3: Stores artifacts and deployment packages.
